# Стадия: In Progress

## Назначение

Каталог задач, находящихся в работе у исполнителя роли Developer.

## Содержание

### Правила

1. Задачу переводит сюда сам исполнитель, беря её из `ready/`; в истории задачи фиксируется, кто взял задачу.
2. Одна задача — один исполнитель; работа ведётся в отдельной ветке по [git-workflow.md](../../docs/development/git-workflow.md).
3. Сюда же возвращаются задачи с ревью при Request changes — с замечаниями в истории задачи.

### Переходы

- **→ `review/`** — изменения готовы, PR создан, чек-лист PR пройден, отчёт добавлен в задачу.
- **→ `blocked/`** — работа невозможна; причина и требуемое решение записаны в задаче.
