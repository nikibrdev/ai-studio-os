# Система задач

## Назначение

Файловая система жизненного цикла задач AI Studio OS. Задача — markdown-файл по шаблону [.claude/templates/Task.md](../.claude/templates/Task.md); стадия задачи определяется каталогом, в котором находится файл.

## Содержание

### Жизненный цикл

Канонический жизненный цикл задачи (9 состояний: Backlog, Ready, In Progress, Review, Testing, Done, Blocked, Cancelled, Archived), диаграмма и правила переходов — [docs/architecture/state-machine.md](../docs/architecture/state-machine.md). Организация процесса и роли — [docs/architecture/workflow.md](../docs/architecture/workflow.md).

**Принято ([ADR-004](../docs/adr/ADR-004-task-storage.md)):** источник истины — PostgreSQL (Task Engine); этот каталог станет markdown-экспортом. До ввода Task Engine действует файловый процесс с отображением состояний на каталоги (переходный период):

| Состояние | Каталог |
| --- | --- |
| Backlog, Ready, In Progress, Review, Blocked, Done | одноимённые каталоги |
| Testing | `review/` (переход фиксируется в «Истории» задачи) |
| Cancelled, Archived | `archive/` (для Cancelled — причина в «Истории») |

### Правила

1. Перемещение задачи между стадиями = перемещение файла между каталогами с записью в разделе «История» задачи.
2. В работу берутся только задачи из `ready/`.
3. Файл задачи — единственное место, где накапливается её история: переходы, замечания, отчёты.
4. Назначение каждой стадии описано в README соответствующего каталога.
