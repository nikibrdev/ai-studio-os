# TASK-085: Живая проверка на реальной инфраструктуре, закрытие EPIC-010

## Тип

test

## Эпик

[EPIC-010 Orchestrator](../../docs/roadmap/EPIC-010-orchestrator.md)

## Цель

Подтвердить вживую (реальный PostgreSQL, реальный Docker) весь механизм: запуск `apps/orchestrator`, бутстрап исполнителя, автоматическое прохождение задачи Ready → In Progress → Review через реальный `agents/claude-code`. Закрыть эпик: критерии, ROADMAP, PROJECT_MANIFEST, PROJECT_HEALTH, CHANGELOG.

## Контекст

Тот же принцип, что TASK-056 (EPIC-006)/TASK-070 (EPIC-008): unit-тесты на фейках не заменяют проверку на настоящей инфраструктуре. Честно принятое ограничение — как и в EPIC-006, реальный `ANTHROPIC_API_KEY` может отсутствовать в среде проверки; в этом случае проверяется механика (переходы состояний, запуск/остановка контейнера, создание ветки, вызов `Accept`/`Finish`), а не качество ответа AI-провайдера — ограничение документируется явно, не замалчивается.

## Scope

### Входит

- Запуск `docker compose up` (PostgreSQL), применение миграций, запуск `apps/orchestrator` (`go run`).
- Через `apps/api` (или напрямую через Application Layer в тестовом сценарии) — создание проекта, подключение репозитория, активация, создание и планирование задачи (`TaskPlanned`).
- Наблюдение: Orchestrator подхватывает событие, создаёт ветку, запускает контейнер, задача переходит в In Progress; по завершении контейнера — либо переход в Review с открытым PR (если AI-провайдер доступен и отработал), либо честно задокументированная остановка на этапе механики (если ключа нет).
- Обновление раздела «Критерии завершения» EPIC-010 (отметки `[x]`), `ROADMAP.md` (прогресс v1.0), `PROJECT_MANIFEST.md`, `PROJECT_HEALTH.md`, `CHANGELOG.md`.

### Не входит

- Открытие следующего эпика декомпозиции v1.0 («Роли PM/QA») — отдельная задача после этой.
