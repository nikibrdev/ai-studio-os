chore

## Эпик

## Цель

Единые редакторские настройки (отступы, кодировка, конец строки) не зависят от личных настроек IDE конкретного разработчика.

## Scope

### Входит

- `.editorconfig`: базовые настройки (UTF-8, LF, финальный перевод строки, обрезка хвостовых пробелов, отступ 2 пробела); `*.go` — табы (согласовано с gofumpt — источник истины для Go всё равно gofumpt, `.editorconfig` лишь синхронизирует поведение редактора до сохранения); `Makefile` — табы (обязательно для синтаксиса make); `*.{bat,cmd,ps1}` — CRLF (согласовано с `.gitattributes`); `*.md` — без обрезки хвостовых пробелов (двойной пробел — способ жёсткого переноса строки в Markdown).

### Не входит

- Правила, которые уже enforced инструментами (gofumpt для Go, markdownlint для остального Markdown-стиля) — `.editorconfig` их не дублирует по содержанию, только синхронизирует базовые параметры редактора.

## Критерии приёмки

- [x] Настройки согласованы с `.gitattributes` (CRLF/LF по тем же группам файлов).
- [x] `.go`-файлы, отформатированные текущим `gofumpt`, не конфликтуют с настройками `.editorconfig` (табы).
- [ ] Прогон verify в PR — зелёный.

## План реализации

Стандартный `.editorconfig` (см. содержание Scope); никакой отдельной проверки/линтера для него не заводится — это подсказка редактору, а не гейт качества (за качество отвечают gofumpt/golangci-lint/markdownlint).
