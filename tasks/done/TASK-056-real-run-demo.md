# TASK-056: Демонстрационный реальный прогон на тестовой задаче

## Тип

feature

## Эпик

[EPIC-006 AI Agent Runtime](../../docs/roadmap/EPIC-006-ai-agent-runtime.md)

## Цель

Провести настоящий прогон адаптера `agents/claude-code` (TASK-055) на одноразовой тестовой задаче: реальный контейнер, реальная (или честно задокументированная как недостижимая в этой сессии) передача секретов вложенному Claude Code, реальный результат — произведённый и опубликованный Artifact.

## Контекст

Пользователь явно выбрал полную целевую модель со реальным прогоном в конце эпика (не макет). Ограничение, зафиксированное при открытии эпика: у текущей сессии нет отдельного Anthropic API-ключа для вложенного процесса; извлечение/переиспользование учётных данных текущей сессии для этой цели не рассматривается — тот же принцип, что и отказ извлекать GitHub-токен через curl ранее в этом проекте. Секреты передаются только явно предоставленным пользователем способом (переменная окружения/примонтированный файл конфигурации), никогда не через извлечение чужого секрета.

## Scope

### Входит

- Подготовка одноразовой тестовой задачи (минимальная, безопасная — например, создать файл или внести тривиальное изменение в тестовом репозитории).
- Реальный запуск: образ TASK-053, жизненный цикл TASK-054, адаптер TASK-055 — сквозь `Accept → Status/Artifacts → Finish`.
- Если секреты для вложенного Claude Code недоступны в этой сессии — честная фиксация предела: что именно проверено (контейнер стартует, рабочая копия клонируется, allowlist работает, секреты git передаются), что не проверено (реальный вызов AI-провайдера) и почему, без имитации успеха.
- Отчёт с фактическим результатом — не гипотетическим.

### Не входит

- Автоматизация этого прогона в CI (реальный прогон — ручная демонстрация, не регрессионный тест; секреты в CI не вносятся без отдельного решения человека, как и в EPIC-005).

## План реализации

1. Спросить пользователя напрямую про секрет вложенного Claude Code (`ANTHROPIC_API_KEY`) — у сессии такого ключа нет, извлекать/переиспользовать учётные данные текущей сессии для этой цели не рассматривается. Пользователь выбрал: пропустить реальный AI-вызов, зафиксировать честный предел.
2. Одноразовая тестовая задача: `platform.ExecutorTask` с `Repository: "nikibrdev/ai-studio-os", Branch: "main"` — тот же безопасный, уже проверенный (TASK-054) объект клонирования; не отдельная новая ветка (реальный AI-вызов всё равно не состоится без ключа, создавать отдельную ветку в реальном репозитории для заведомо неполного прогона нецелесообразно).
3. Реальный прогон целиком: `Executor.Accept` (образ TASK-053 + жизненный цикл TASK-054, реальный `GIT_TOKEN` из `git credential fill`, `ProviderAPIKey=""`) → цикл `Status` → `Artifacts` → `Finish`, через одноразовую Go-программу вне репозитория (`tmp-demo056/`, удалена по завершении — это ручная демонстрация, не автоматизированный тест, по scope задачи).
4. **Первый прогон вскрыл реальный дефект**: `Status` дошёл до `failed` (ожидаемо — `claude` без `ANTHROPIC_API_KEY` завершается с ненулевым кодом), но `Artifacts` немедленно упал с «container ... is not running» — контейнер уже полностью вышел к моменту вызова. Причина: `cloneAndRunScript` (TASK-054) исполняет команду как замену PID 1 — процесс завершается вместе с контейнером мгновенно после команды, не оставляя окна для `docker exec`. Исправлено: скрипт теперь захватывает код возврата в файл (`exitCodeFile`) и засыпает 300 секунд вместо немедленного выхода; `Status` читает файл вместо `docker inspect .State.Running` (последний перестал быть информативным, раз контейнер всегда «жив» во время сна).
5. **Второй прогон вскрыл второй реальный дефект**: `Artifacts` подключился к контейнеру, но `git log` упал с «not a git repository» — `docker exec` без явного `--workdir` выполняется в `WORKDIR` образа (`/workspace`), а не в клоне (`/workspace/repo`). Исправлено: `Exec` теперь передаёт `--workdir /workspace/repo` (константа `workspaceDir`, общая с `script.go`, чтобы не разойтись).
6. После обоих исправлений — третий прогон: `Accept` → `Status` (running → running → failed) → `Artifacts` (20 реальных коммитов из истории репозитория) → `Finish` — весь цикл корректен. Повторён ещё раз для стабильности; после каждого прогона — ноль оставшихся контейнеров/сетей.
7. Юнит-тесты `container`-пакета обновлены под новую механику `Status` (файл вместо `docker inspect`) и `Exec` (`--workdir`); добавлены прямые тесты на `cloneAndRunScript`'а новую структуру (захват кода возврата, сон).
8. README `agents/claude-code` — раздел про обнаруженный и исправленный дефект «контейнер не завершается сразу после команды».
