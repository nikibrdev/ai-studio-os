# TASK-059: Файловое хранилище Memory (durable-часть)

## Тип

feature

## Эпик

[EPIC-007 Memory System](../../docs/roadmap/EPIC-007-memory-system.md)

## Цель

Реализовать durable-хранилище знаний по формату из TASK-058: запись и чтение файлов `memory/<projectID>/<id>.md`, независимо от Qdrant (Qdrant — производный индекс поверх этого хранилища, TASK-060/061).

## Контекст

`memory/` в корне репозитория — данные, не код ([module-boundaries.md](../../docs/architecture/module-boundaries.md)); реализация — `internal/infrastructure/memory`, отдельный Go-пакет, читающий/пишущий в эту директорию во время выполнения.

## Scope

### Входит

- `internal/infrastructure/memory/filestore.go` — `Write(entry)`/`Read(projectID, id)`/`List(projectID)` (или аналог — уточнить в плане) поверх формата TASK-058.
- Конкретный тип `Entry`, реализующий интерфейс `platform.MemoryEntry` (ID/ProjectID/Kind/Content/Source/RecordedAt).
- Тесты на реальной файловой системе (временный каталог, `t.TempDir()`) — не фейк, файловые операции детерминированы и быстры без Docker.

### Не входит

- Qdrant, эмбеддинги (TASK-060).
- Полный `MemoryProvider` (TASK-061) — эта задача даёт только durable-часть, не публичный контракт целиком.

## План реализации

1. `Entry` — конкретный тип, реализующий `platform.MemoryEntry`; `NewEntry(projectID, kind, content, source)` генерирует ID и `RecordedAt`; `restoreEntry(...)` — точечная реконструкция без бизнес-правил (тот же принцип, что `Restore` в доменных пакетах EPIC-005), нужна при чтении из файла.
2. ID — UUID v4 из `crypto/rand`, без внешней библиотеки (тот же принцип, что `internal/application/id.go`) — выбран именно UUID-формат (не произвольный hex), потому что ADR-018 требует его как формат точки Qdrant; один и тот же ID используется и как имя файла, и как ID точки (TASK-061), не две разные схемы.
3. `FileStore{rootDir}` — `Write`/`Read`/`List` поверх формата TASK-058; сериализация/парсинг frontmatter — простой ручной разбор (`key: value` построчно + разделитель `---`), без YAML-библиотеки (формат простой, YAML не нужен).
4. Защита от path traversal — `projectID`/`id`, содержащие `/`, `\` или `..`, отклоняются (`ErrInvalidID`) до построения пути; хотя ID сейчас всегда генерируются самим пакетом (доверенный формат), значения приходят через публичный API (`Read`/`List`), где нельзя полагаться на то, что вызывающая сторона гарантированно передаёт уже провалидированный ID.
5. Тесты на `t.TempDir()`: round-trip (включая кириллицу и многострочное содержимое), перезапись, изоляция по проектам, пустой список для незаписанного проекта (не ошибка), защита от path traversal.
