# Стадия: Blocked

## Назначение

Каталог задач, работа над которыми невозможна до получения внешнего решения или снятия зависимости.

## Содержание

### Критерий попадания

В истории задачи зафиксировано: **причина блокировки** и **какое решение или событие требуется** для продолжения. Блокировка без записанной причины не допускается.

### Типичные причины

- Требуется архитектурное решение (Open Question → ADR).
- Зависимость от другой незавершённой задачи.
- Отсутствует доступ, данные или инфраструктура.

### Правила

1. Перевести задачу в blocked может любая роль, столкнувшаяся с блокировкой.
2. Разбор блокировок — обязанность Project Manager: он организует получение решения.
3. Blocked-задачи регулярно пересматриваются; «вечных» блокировок быть не должно.

### Переходы

- **→ `in-progress/`** или **→ `ready/`** — блокировка снята (решение получено); факт снятия записан в истории.
- **→ `archive/`** — задача потеряла актуальность.
