# Стадия: Review

## Назначение

Каталог задач, ожидающих или проходящих проверку: ревью кода (Reviewer) и контроль качества (QA Engineer). В переходный период [ADR-004](../../docs/adr/ADR-004-task-storage.md) (до ввода Task Engine) здесь же находятся задачи в состоянии Testing — переход Review → Testing фиксируется в «Истории» задачи (см. [state-machine.md](../../docs/architecture/state-machine.md)).

## Содержание

### Критерий попадания

PR создан и оформлен по шаблону; чек-лист [PR.md](../../.claude/checklists/PR.md) пройден автором; отчёт о выполнении добавлен в файл задачи.

### Правила

1. Ревью проводится по [docs/development/review-process.md](../../docs/development/review-process.md).
2. QA-проверка — по чек-листу [QA.md](../../.claude/checklists/QA.md) (для этапов, где QA включён в процесс).
3. Итог проверки фиксируется в PR и в истории задачи.

### Переходы

- **→ `done/`** — ревью одобрено, PR слит, Definition of Done выполнен.
- **→ `in-progress/`** — Request changes; замечания записаны в задаче.
- **→ `blocked/`** — проверка выявила вопрос, требующий внешнего решения.
