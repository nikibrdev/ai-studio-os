# TASK-020: Релизные заметки и процесс тегирования

## Тип

chore

## Эпик

EPIC-002.5 ([docs/roadmap/EPIC-002.5-engineering-platform.md](../../docs/roadmap/EPIC-002.5-engineering-platform.md)) — последняя задача блока.

## Цель

Категоризация GitHub release notes по типам Conventional Commits; зафиксирован процесс тегирования и соотношение GitHub Release notes ↔ CHANGELOG.md.

## Контекст

По плану эпика (TASK-020). GitHub категоризирует автосгенерированные release notes по **меткам PR**, не по префиксу коммита — до этой задачи PR никак не помечались, поэтому категоризация была бы пустой (все PR попадали бы в «Other Changes»). Заодно обнаружен пробел: TASK-012…019 (публикация репозитория, CI, CODEOWNERS, шаблоны, защита ветки, markdownlint, commit-lint, dependabot) не были отражены в CHANGELOG.md — исправляю в этой же задаче, чтобы не оставлять след без источника истины.

## Scope

### Входит

- `.github/release.yml` — категории по 7 меткам типов (`feat`/`fix`/`docs`/`refactor`/`test`/`chore`/`ci`), fallback «Other Changes».
- 7 меток типов созданы в репозитории; PR #1–8 размечены задним числом по фактическому типу их коммитов.
- `docs/development/git-workflow.md`: правило PR (п. 4) — метка типа обязательна; новый раздел «Релизный процесс» (SemVer-тегирование, соотношение GitHub Release notes и CHANGELOG.md); Decision Required уточнён (текущая практика — merge commit, до ADR-008).
- `.claude/checklists/PR.md` — пункт про метку типа.
- `CHANGELOG.md` — восполнены записи за TASK-012…019 (публикация, CI, CODEOWNERS, шаблоны, защита main, markdownlint, commit-lint, dependabot) + запись за TASK-020.

### Не входит

- Формальное тегирование первого релиза (рано — Domain Layer ещё не начат); проставление меток на будущие PR — процесс, не единоразовое действие.

## Критерии приёмки

- [x] `.github/release.yml` — валиден (js-yaml).
- [x] 7 меток существуют в репозитории; PR #1–8 размечены.
- [ ] Прогон verify в PR — зелёный.
- [ ] CHANGELOG.md отражает TASK-012…020 (проверка чтением).

## План реализации

1. Создать метки типов через API (цвета по распространённой конвенции).
2. Разметить смерженные PR #1–8 по их фактическому commit-типу (7 → `chore`, PR #7 → `ci`, единственный использованный тип `ci` к моменту его PR).
3. `.github/release.yml` с категориями = меткам.
4. Дополнить git-workflow.md и чек-лист PR.
5. Восполнить CHANGELOG.md за пропущенный блок TASK-012…019 и добавить запись TASK-020.
