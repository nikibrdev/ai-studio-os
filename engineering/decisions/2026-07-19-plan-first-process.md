# Решение: процесс «план → утверждение → код → ревью → merge»

## Назначение

Фиксирует переход проекта с режима «Claude как генератор» на инженерный процесс, в котором AI-агент работает как сотрудник команды.

## Содержание

### Контекст

До EPIC-002 включительно Claude выполнял крупные поручения целиком. Ревью EPIC-002 показало: без промежуточного утверждения агент принимает микрорешения (контракт Agent, имя Engine, структура каталогов), которые приходится откатывать.

### Решение (принял архитектор проекта, 2026-07-19)

Цикл работы над каждой задачей:

1. Человек пишет задачу (TASK).
2. Claude пишет **план реализации** в файл задачи и останавливается.
3. Человек проверяет и утверждает план.
4. Claude пишет код строго по плану.
5. Человек делает code review.
6. Claude исправляет замечания.
7. Merge.

Дополнительно: один Epic = 5–15 малых задач; одна задача — не более одного модуля; после каждой задачи проект собирается и проходит `make verify`.

### Последствия для процесса

- [CLAUDE.md](../../CLAUDE.md) и [.claude/context/rules.md](../../.claude/context/rules.md) обновлены (план-этап обязателен).
- Шаблон [Task.md](../../.claude/templates/Task.md) дополнен разделом «План реализации».
- Итоги ревью фиксируются в [reviews/](../reviews/).
