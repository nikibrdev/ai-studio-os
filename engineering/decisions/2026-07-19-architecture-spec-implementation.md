# Решение: цикл Architecture → Specification → Implementation

## Назначение

Фиксирует стратегическое решение архитектора проекта (2026-07-19) о порядке разработки после EPIC-002.5 и EPIC-002.6.

## Содержание

### Контекст

До сих пор проект строил архитектуру. С началом реализации нужен цикл, сохраняющий контроль при росте до десятков тысяч строк и десятков AI-агентов.

### Решение

Разработка ведётся по циклу:

1. **Architecture** — принимается архитектурное решение (ADR).
2. **Specification** — для каждого модуля пишется короткая спецификация: что он должен делать и чего не должен.
3. **Implementation** — только после этого Claude Code получает задачу на реализацию.

Дополняет (не заменяет) процесс задач «план → утверждение → код → ревью → merge» ([2026-07-19-plan-first-process.md](2026-07-19-plan-first-process.md)): спецификация — вход задачи, план — способ её выполнения.

### Последствия для процесса

- Задача на реализацию модуля без спецификации не соответствует Definition of Ready.
- За основу спецификации берётся шаблон [Module.md](../../.claude/templates/Module.md) (контракт, события, зависимости, ограничения и «чего модуль НЕ делает»).

### Открытый вопрос

Место хранения спецификаций: раздел в README модуля или отдельные файлы (например, `docs/architecture/specs/`) — уточнить при первой спецификации (Domain Layer).

### Дополнение 2026-07-19: вопрос решён архитектором

Приняты **оба уровня с разными ролями**:

- **README в модуле** — краткая документация разработчика (1–2 страницы): назначение, зависимости, экспортируемые сущности, ограничения, ссылки.
- **Specification в `docs/specifications/<слой>/<модуль>.md`** — полноценное ТЗ: требования, сценарии, инварианты, события, ограничения, будущие расширения. Каталог: `docs/specifications/{domain,application,platform,infrastructure}`. Именно отсюда Claude Code берёт требования перед реализацией.

Введено **правило нового пакета**: пакет не появляется без README + Specification + TASK + Acceptance Criteria ([docs/specifications/README.md](../../docs/specifications/README.md)); шаблон — [Specification.md](../../.claude/templates/Specification.md).
