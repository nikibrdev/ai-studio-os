# Решение: запрет прямых коммитов в main — без исключений

## Назначение

Фиксирует правило архитектора проекта (2026-07-19): все изменения проходят через ветку и PR, даже при работе в одиночку.

## Содержание

### Правило

С момента публикации репозитория на GitHub прямые коммиты в `main` запрещены **без исключений**, даже если разработчик работает один:

```
изменение → feature branch → PR → Review → Merge
```

Ревью может выполнять Claude (агент-ревьюер допустим для процессной дисциплины); это не отменяет открытых подвопросов [ADR-008](../../docs/adr/ADR-008-git-policies.md) о числе обязательных ревьюеров и участии человека — до его принятия действует минимум: одно ревью на PR.

### Зачем

- Дисциплина процесса не зависит от числа участников.
- Каждый merge проходит одинаковые проверки (`make verify` в CI) — люди и агенты в равных условиях.
- История PR — источник метрик (замечания ревью, время задачи).

### Область действия

- Бутстрап-коммиты до публикации репозитория (d88f2eb…) — единовременное исключение фазы инициализации; после push правило действует безусловно.
- Техническое закрепление — защита ветки (EPIC-002.5, TASK-016).
