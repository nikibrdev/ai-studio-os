# Решение: GitHub-репозиторий проекта

## Назначение

Фиксирует решения архитектора проекта (2026-07-19) по размещению репозитория — ответы на вопросы EPIC-002.5.

## Содержание

### Решения

1. **Имя:** `ai-studio-os` — без приставок; будущие SDK именуются единообразно: `ai-studio-go`, `ai-studio-js`, `ai-studio-cli`.
2. **Видимость:** **Public** — полноценная защита веток, GitHub Actions без ограничений для open-source, внешние контрибуции, портфолио. Закрытые модули при необходимости — отдельными приватными репозиториями.
3. **Владелец:** личный аккаунт мейнтейнера (без организации). Организация — позже, когда появятся сайт, CLI, SDK, несколько репозиториев; сейчас она добавит только административную сложность.
4. **Порядок создания:** пустой репозиторий через веб-интерфейс GitHub (без README/LICENSE/.gitignore — уже есть в проекте), затем:

```
git init
git branch -M main
git remote add origin <URL_репозитория>
git add .
git commit -m "chore: bootstrap AI Studio OS"
git push -u origin main
```

Защита ветки и GitHub Actions настраиваются после первого push (EPIC-002.5, TASK-013/016).

### Следствия

- Публичность повышает требования к LICENSE — решение по [ADR-001](../../docs/adr/ADR-001-license.md) становится срочным.
- Имя Go-модуля будет заменено на `github.com/<owner>/ai-studio-os` отдельной задачей после публикации ([ADR-009](../../docs/adr/ADR-009-toolchain.md)).
