# Решение: порты хранения агрегатов объявляются в internal/application, не в internal/platform

## Назначение

Фиксирует решение архитектора (2026-07-21, открытие EPIC-004) о размещении интерфейсов хранения доменных агрегатов (`ProjectStore`, `TaskStore`, `ExecutorStore`, `ExecutionStore`, `ArtifactStore`).

## Содержание

### Контекст

`internal/platform` — платформенные абстракции, сознательно домен-независимые ([ADR-015](../../docs/adr/ADR-015-internal-layering.md)): `EventBus`, `Tool`, `MemoryProvider`, `RepositoryProvider`, `Executor` не знают о конкретных доменных типах (Task, Project и т.д.), только об абстрактных плейсхолдерах или примитивах. Application Layer (v0.4, EPIC-004), напротив, оперирует конкретными доменными агрегатами — use-case'ам нужно их загружать и сохранять.

### Решение

Порты хранения (`Get`/`Save` по каждому агрегату) объявляются как интерфейсы в `internal/application`, рядом с use-case'ами, которые их используют (driven ports, гексагональная архитектура) — не в `internal/platform`. Причина: сигнатуры портов неизбежно содержат доменные типы (`*task.Task`, `*project.Project` и т.д.); поместить их в `internal/platform` означало бы протащить в домен-независимый слой знание о конкретных доменных сущностях — прямое нарушение ADR-015.

Инфраструктурные реализации портов (PostgreSQL, EPIC-005) размещаются в `internal/infrastructure` и подключаются к `internal/application` тем же способом, каким `internal/platform.EventBus` подключается к своей in-memory реализации — через внедрение зависимости при сборке приложения, не импортом в обратную сторону.

Для тестов этого эпика — `internal/application/inmemory`: детерминированные фейки без персистентности, не претендующие на роль инфраструктурного адаптера.

### Границы решения

1. Это решение — размещение, не форма контракта: сигнатуры `Get(ctx, id) (*T, error)`/`Save(ctx, *T) error` могут уточняться по мере появления реальных use-case'ов (TASK-041…045) без пересмотра самого размещения.
2. Не отменяет и не изменяет ADR-015 — наоборот, следует из него буквально.
3. Формат идентификаторов, используемых в качестве ключей портов, — уже решён [ADR-011](../../docs/adr/ADR-011-task-identifiers.md) (строки).
