# Engineering: Metrics

## Назначение

Метрики инженерного процесса: скорость прохождения задач по стадиям, доля возвратов с ревью/тестирования, повторяемость замечаний. Наполняется по мере появления данных (после ввода Task Engine метрики будут считаться из событий).

## Содержание

### Снимки

Снимки метрик генерируются командой `make metrics` (скрипт [scripts/metrics.sh](../../scripts/metrics.sh)) в файлы `YYYY-MM-DD.md`. Снимок не редактируется после создания — каталог исключён из `make md-lint` (TASK-017), чтобы автофиксы линтера не переписывали зафиксированные данные.

### Что собирается автоматически

Количество ADR (по статусам), задачи по стадиям, Go-пакеты, покрытие пакетов README, количество спецификаций, TODO в коде, число markdown-документов.

### Что появится позже

Среднее время задачи и среднее число замечаний ревью — после истории PR в GitHub и событий Task Engine. Автозапуск снимков в CI — EPIC-002.5+.
