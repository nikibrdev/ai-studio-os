# Engineering

## Назначение

Инженерная летопись проекта: ревью, ретроспективы, процессные решения и метрики. Через год здесь будет вся история того, как проект строился и почему. Архитектурные решения сюда не входят — они в [docs/adr/](../docs/adr/).

## Содержание

| Каталог | Что хранит |
| --- | --- |
| [reviews/](reviews/) | Записи code review: замечания, решения, исправления |
| [retrospective/](retrospective/) | Ретроспективы эпиков и релизов: что сработало, что менять |
| [decisions/](decisions/) | Процессные (не архитектурные) решения: как работаем |
| [metrics/](metrics/) | Метрики инженерного процесса |

### Правила

1. Записи не редактируются задним числом — только дополняются новыми.
2. Формат имени файла: `YYYY-MM-DD-<short-name>.md`.
3. Каждая запись — по единой структуре документов проекта.
