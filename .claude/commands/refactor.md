# Команда: refactor

## Назначение

Шаблон команды для рефакторинга — улучшения структуры кода без изменения поведения. Используется ролью Developer только по отдельной задаче на рефакторинг.

## Содержание

### Предусловия

- Задача в `tasks/ready/` с явной целью рефакторинга и границами затрагиваемого кода.
- Поведение затрагиваемого кода закреплено тестами, либо задача явно определяет способ проверки неизменности поведения.

### Входные данные

- Цель рефакторинга (какая проблема структуры устраняется).
- Границы: какие пакеты/модули затрагиваются.

### Шаги

1. Убедиться, что поведение закреплено (тесты проходят до изменений).
2. Выполнять рефакторинг малыми шагами; каждый шаг — отдельный осмысленный коммит (`refactor: ...`).
3. После каждого шага проверять, что поведение не изменилось (тесты/проверки проходят).
4. Не смешивать рефакторинг с новой функциональностью и исправлениями.
5. Обновить документацию, если изменились имена или структура публичных контрактов (изменение самих контрактов — не рефакторинг и требует отдельной задачи/ADR).
6. Оформить PR и отчёт по стандартному процессу.

### Результат

Структура кода улучшена, поведение не изменилось, история изменений читается по шагам; PR готов к ревью.
