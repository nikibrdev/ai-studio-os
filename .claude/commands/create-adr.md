# Команда: create-adr

## Назначение

Шаблон команды для оформления архитектурного решения (ADR). Выполняется ролью Architect или, **по явному поручению**, другой ролью. AI-агент не инициирует ADR самостоятельно.

## Содержание

### Предусловия

- Задача в `tasks/ready/` с явным поручением оформить ADR и с сутью решаемого вопроса.
- Вопрос действительно архитектурный (см. критерии в [CONSTITUTION.md](../../CONSTITUTION.md) и [CLAUDE.md](../../CLAUDE.md)).

### Входные данные

- Формулировка проблемы и контекст.
- Рассматриваемые варианты (если уже известны).
- Ограничения (стек, принципы, зависимости).

### Шаги

1. Определить номер: следующий по сквозной нумерации в `docs/adr/`.
2. Создать файл `docs/adr/ADR-NNN-short-name.md` по шаблону [ADR.md](../templates/ADR.md).
3. Описать контекст: проблема, требования, почему решение нужно сейчас.
4. Описать не менее двух вариантов с плюсами и минусами.
5. Зафиксировать решение и обоснование (если решение принимает человек — оставить статус «Предложено» и передать на утверждение).
6. Описать последствия: положительные, отрицательные, влияние на документы и код.
7. Обновить документы, на которые влияет решение (после принятия).
8. Оформить PR (`docs: ...`) и отчёт по стандартному процессу.

### Результат

ADR в `docs/adr/` со статусом «Предложено» или «Принято»; связанные документы обновлены или перечислены к обновлению.
