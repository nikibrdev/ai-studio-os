# Команда: review

## Назначение

Шаблон команды для проведения ревью Pull Request. Используется ролью Reviewer; процесс — [docs/development/review-process.md](../../docs/development/review-process.md).

## Содержание

### Предусловия

- PR оформлен по шаблону, автор прошёл чек-лист [PR.md](../checklists/PR.md).
- Задача находится в `tasks/review/`.

### Входные данные

- Ссылка на PR и файл задачи.

### Шаги

1. Прочитать задачу: цель, scope, критерии приёмки.
2. Проверить соответствие изменений scope задачи (нет лишних изменений).
3. Проверить соблюдение архитектурных принципов и границ модулей ([CONSTITUTION.md](../../CONSTITUTION.md)).
4. Проверить соответствие стандартам кода ([coding-standard.md](../../docs/development/coding-standard.md)).
5. Проверить обновление документации и запись в CHANGELOG (если требуется).
6. Проверить отсутствие несанкционированных архитектурных решений; при обнаружении — потребовать вынесения в Open Questions/ADR.
7. Сформулировать замечания: конкретно, с разделением на блокирующие и рекомендации.
8. Вынести решение: Approve или Request changes; зафиксировать итог в PR и файле задачи.

### Результат

Обоснованное решение по PR; замечания зафиксированы; задача переведена дальше по процессу (done или возврат в in-progress).
