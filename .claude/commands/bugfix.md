# Команда: bugfix

## Назначение

Шаблон команды для исправления дефекта. Используется ролью Developer по задаче-дефекту, оформленной QA Engineer или Project Manager.

## Содержание

### Предусловия

- Задача-дефект в `tasks/ready/` с шагами воспроизведения, ожидаемым и фактическим поведением.

### Входные данные

- Шаги воспроизведения.
- Ожидаемое поведение (со ссылкой на спецификацию/документацию).
- Фактическое поведение.

### Шаги

1. Воспроизвести дефект по шагам из задачи.
2. Локализовать причину; зафиксировать её в задаче (причина, а не только симптом).
3. Написать тест, воспроизводящий дефект (если тестирование входит в scope этапа) — тест должен падать до исправления.
4. Исправить причину минимальным изменением; не рефакторить попутно.
5. Убедиться, что тест проходит и регрессии нет.
6. Обновить документацию, если дефект был вызван её неточностью.
7. Оформить PR (`fix: ...`) и отчёт; в отчёте указать причину дефекта.

### Результат

Дефект исправлен по причине, а не по симптому; воспроизводящий тест закреплён; PR готов к ревью.
