# Команда: create-module

## Назначение

Шаблон команды для создания нового модуля ядра. Используется ролью Developer по задаче, подготовленной Project Manager и одобренной Architect.

## Содержание

### Предусловия

- Задача в `tasks/ready/` с описанием модуля и его границ.
- Модуль и его место в структуре утверждены (при необходимости — ADR).

### Входные данные

- Имя модуля и назначение.
- Публичный контракт (интерфейсы) модуля.
- Зависимости модуля (только через интерфейсы).

### Шаги

1. Прочитать обязательные документы ([CLAUDE.md](../../CLAUDE.md)) и задачу.
2. Создать документацию модуля по шаблону [Module.md](../templates/Module.md) (Documentation First).
3. Определить публичные интерфейсы модуля (Interface First).
4. Реализовать модуль в согласованном месте структуры, не нарушая границ (`internal/`, зависимости внутрь).
5. Покрыть доменную логику unit-тестами (если тестирование входит в scope этапа).
6. Обновить [docs/architecture/components.md](../../docs/architecture/components.md).
7. Оформить PR и отчёт по стандартному процессу.

### Результат

Модуль с документацией и контрактом, не нарушающий архитектурных границ; PR готов к ревью.
