# Команда: create-api

## Назначение

Шаблон команды для создания или расширения API. Используется ролью Developer после того, как контракт API спроектирован и утверждён.

## Содержание

### Предусловия

- Задача в `tasks/ready/` со спецификацией эндпоинтов/операций.
- Протокол и стиль API определены (на этапе Foundation — открытый вопрос; команда применима после его решения).

### Входные данные

- Спецификация API по шаблону [API.md](../templates/API.md): операции, входные/выходные данные, ошибки.
- Требования к авторизации (если определены).

### Шаги

1. Прочитать обязательные документы и задачу.
2. Зафиксировать/обновить спецификацию в `docs/api/` (Documentation First).
3. Реализовать API-слой в `apps/api/`, вызывая доменную логику `internal/` через интерфейсы; без бизнес-логики в хендлерах.
4. Обработать ошибки согласно принятому в спецификации формату.
5. Написать тесты API (если тестирование входит в scope этапа).
6. Оформить PR и отчёт по стандартному процессу.

### Результат

Работающий API-слой, точно соответствующий спецификации в `docs/api/`; PR готов к ревью.
