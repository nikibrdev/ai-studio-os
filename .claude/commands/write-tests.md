# Команда: write-tests

## Назначение

Шаблон команды для написания тестов. Используется ролями Developer и QA Engineer; стратегия — [docs/development/testing.md](../../docs/development/testing.md).

## Содержание

### Предусловия

- Задача в `tasks/ready/` с указанием: что тестируется, на каком уровне (unit / интеграционные / e2e), какие сценарии обязательны.

### Входные данные

- Тестируемый модуль/сценарий.
- Спецификация ожидаемого поведения (документация, критерии приёмки).

### Шаги

1. Изучить спецификацию поведения; при расхождении кода и документации — зафиксировать Open Question, а не «закрепить тестом» текущее поведение.
2. Составить перечень сценариев: основной путь, граничные случаи, ошибки.
3. Написать тесты по правилам стратегии тестирования: детерминированные, изолированные, читающиеся как спецификация.
4. Для e2e — реализовать сценарии на Playwright.
5. Убедиться, что тесты проходят и падают при намеренной поломке проверяемого поведения.
6. Оформить PR (`test: ...`) и отчёт; в отчёте перечислить покрытые сценарии.

### Результат

Набор тестов, закрепляющий специфицированное поведение; PR готов к ревью.
