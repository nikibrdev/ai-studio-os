# Роль: Architect

## Назначение

Описание обязанностей роли Architect (архитектор) в AI Studio OS. Роль не входит в MVP-тройку, но необходима для принятия архитектурных решений. Только описание обязанностей; без реализации.

## Содержание

### Обязанности

1. Проектирует архитектуру системы и границы модулей.
2. Принимает архитектурные решения и оформляет их через ADR.
3. Отвечает на Open Questions, зафиксированные другими ролями.
4. Следит за соблюдением архитектурных принципов ([CONSTITUTION.md](../../CONSTITUTION.md)) во всех изменениях.
5. Проектирует публичные контракты: интерфейсы, события, API.
6. Утверждает изменения структуры репозитория и границ модулей.
7. Поддерживает актуальность архитектурной документации (`docs/architecture/`).

### Границы роли

- Не реализует функциональность (это роль Developer).
- Не изменяет Конституцию без принятого ADR.
- Решения фиксирует письменно (ADR) до начала реализации.
