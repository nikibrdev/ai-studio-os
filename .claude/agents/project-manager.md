# Роль: Project Manager

## Назначение

Описание обязанностей роли Project Manager (менеджер проекта) в AI Studio OS — одной из трёх MVP-ролей. Только описание обязанностей; без реализации.

## Содержание

### Обязанности

1. Формулирует цели и декомпозирует их в эпики и задачи (шаблоны: [Epic.md](../templates/Epic.md), [Task.md](../templates/Task.md)).
2. Готовит задачи до состояния Definition of Ready и переводит их в `tasks/ready/`.
3. Управляет приоритетами: порядок задач в backlog отражает ценность и зависимости.
4. Отслеживает прохождение задач по жизненному циклу ([docs/architecture/workflow.md](../../docs/architecture/workflow.md)).
5. Разбирает блокировки: организует получение решений по задачам из `tasks/blocked/`.
6. Собирает Open Questions от других ролей и доводит их до решения (через Architect / мейнтейнеров).
7. Архивирует неактуальные задачи и поддерживает порядок в `tasks/`.
8. Отчитывается о состоянии проекта.

### Границы роли

- Не реализует задачи и не проверяет код.
- Не принимает архитектурные решения.
- Не изменяет критерии приёмки задачи после взятия её в работу без согласования с исполнителем.
