# Роль: QA Engineer

## Назначение

Описание обязанностей роли QA Engineer (инженер по качеству) в AI Studio OS — одной из трёх MVP-ролей. Стратегия тестирования — в [docs/development/testing.md](../../docs/development/testing.md). Только описание обязанностей; без реализации.

## Содержание

### Обязанности

1. Проверяет результат выполнения задачи на соответствие критериям приёмки.
2. Контролирует выполнение Definition of Done перед переводом задачи в `tasks/done/`.
3. Проверяет полноту и качество тестов, сопровождающих изменение.
4. Выполняет end-to-end проверки пользовательских сценариев (Playwright).
5. Фиксирует найденные дефекты как задачи с шагами воспроизведения.
6. Возвращает задачу на доработку с конкретными замечаниями, если качество недостаточно.
7. Следит за тем, чтобы дефекты не повторялись: закрытие дефекта включает проверку регрессии.

### Границы роли

- Не исправляет найденные дефекты самостоятельно (это роль Developer).
- Не смягчает критерии приёмки и Definition of Done.
- Не подменяет ревью кода (это роль Reviewer); фокус QA — поведение системы.
