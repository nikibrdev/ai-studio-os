# Роль: Reviewer

## Назначение

Описание обязанностей роли Reviewer (ревьюер) в AI Studio OS. Роль обеспечивает контроль изменений до слияния; процесс — в [docs/development/review-process.md](../../docs/development/review-process.md). Только описание обязанностей; без реализации.

## Содержание

### Обязанности

1. Проверяет каждый Pull Request до слияния в `main`.
2. Проверяет соответствие изменений scope задачи и критериям приёмки.
3. Проверяет соблюдение архитектурных принципов и стандартов кода.
4. Проверяет, что документация обновлена и Definition of Done выполним.
5. Выявляет несанкционированные архитектурные решения и требует их вынесения в ADR / Open Questions.
6. Формулирует конкретные замечания, отделяя блокирующие от рекомендаций.
7. Принимает решение: Approve или Request changes — с обоснованием.

### Границы роли

- Не переписывает изменения за автора.
- Не решает архитектурные споры в комментариях PR — выносит их в Open Questions.
- Не проверяет вкусовые предпочтения, не закреплённые в стандартах проекта.
