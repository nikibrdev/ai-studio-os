# Чек-лист: Pull Request

## Назначение

Самопроверка автора (человека или агента) перед запросом ревью. Пройденный чек-лист — условие перевода задачи в `tasks/review/`.

## Содержание

### Scope и оформление

- [ ] PR решает ровно одну задачу из `tasks/`; ссылка на файл задачи есть.
- [ ] Scope не расширен: нет «попутных» исправлений и рефакторинга.
- [ ] Шаблон PR заполнен полностью.
- [ ] Ветка и коммиты именованы по [git-workflow.md](../../docs/development/git-workflow.md).

### Качество

- [ ] Код соответствует [coding-standard.md](../../docs/development/coding-standard.md).
- [ ] Форматирование и статический анализ пройдены (для этапов, где они включены).
- [ ] Тесты написаны и проходят (для этапов, где тестирование в scope).
- [ ] Архитектурные границы не нарушены; несанкционированных решений нет.

### Документация

- [ ] Затронутая документация обновлена в этом PR; поля «Последнее обновление» актуальны.
- [ ] CHANGELOG.md дополнен (если изменение видимо пользователям).
- [ ] Open Questions зафиксированы (если возникли).

### Процесс

- [ ] Файл задачи перемещён в `tasks/review/`; история задачи дополнена.
- [ ] Отчёт о выполнении добавлен в файл задачи.
