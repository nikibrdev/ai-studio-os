# Чек-лист: QA

## Назначение

Проверка результата задачи ролью QA Engineer перед переводом в `tasks/done/`. Дополняет ревью кода проверкой поведения системы.

## Содержание

### Соответствие задаче

- [ ] Все критерии приёмки из файла задачи выполнены и проверены.
- [ ] Поведение соответствует документации; расхождения зафиксированы как дефекты или Open Questions.
- [ ] Ничего за пределами scope не сломано (проверена смежная функциональность).

### Тесты

- [ ] Тесты покрывают основной путь, граничные случаи и ошибки.
- [ ] Тесты детерминированы и проходят стабильно.
- [ ] End-to-end сценарии (Playwright) проходят — для изменений с пользовательскими сценариями.

### Дефекты

- [ ] Найденные дефекты оформлены задачами с шагами воспроизведения.
- [ ] Блокирующие дефекты возвращают задачу в `in-progress` с записью в истории задачи.

### Завершение

- [ ] Definition of Done ([CONSTITUTION.md](../../CONSTITUTION.md)) выполнен полностью.
- [ ] Задача перемещена в `tasks/done/`; итог зафиксирован в истории задачи.
