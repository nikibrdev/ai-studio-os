# Шаблон: Specification

## Назначение

Шаблон полной спецификации модуля — технического задания для реализации. Хранится в `docs/specifications/<слой>/<модуль>.md`; краткий README модуля ссылается на неё. Правила — [docs/specifications/README.md](../../docs/specifications/README.md).

## Содержание

```markdown
# Спецификация: <слой>/<модуль>

## Назначение

<Одним абзацем: зачем модуль существует и какую ответственность несёт.>

## Содержание

### Требования

<Нумерованный список: что модуль ДОЛЖЕН делать (SPEC-NNN-1, SPEC-NNN-2, ...).>

### Чего модуль НЕ делает

<Явные границы: смежные ответственности, которые модулю запрещены.>

### Сценарии использования

<Основные сценарии: актор → действие → результат. Включая ошибки.>

### Инварианты

<Утверждения, которые модуль гарантирует всегда.>

### События

<Публикуемые: имя, когда возникает, данные. Потребляемые: имя, реакция.>

### Ограничения

<Архитектурные (ADR, границы), технологические, производительность.>

### Будущие расширения

<Что предполагается позже; что в дизайне нужно не закрыть.>

### Acceptance Criteria

- [ ] <Проверяемый критерий приёмки реализации>

## Статус

<Черновик | Утверждена | Реализована>

## Последнее обновление

YYYY-MM-DD
```
