# Шаблон: Module

## Назначение

Шаблон документации модуля ядра. Заполняется до реализации модуля (Documentation First) командой [create-module](../commands/create-module.md).

## Содержание

```markdown
# Модуль: <имя>

## Назначение

<Единственная ответственность модуля. Какую часть домена он реализует.>

## Содержание

### Публичный контракт

<Интерфейсы, предоставляемые модулем: имя, назначение, основные методы.>

### Публикуемые события

<События, которые модуль публикует: имя, когда возникает, состав данных.>

### Потребляемые события

<События других модулей, на которые модуль подписан.>

### Зависимости

<Интерфейсы (порты), которые модуль требует от инфраструктуры или других модулей.
Прямые зависимости от других доменных модулей запрещены.>

### Ограничения и инварианты

<Правила, которые модуль гарантирует и требует.>

## Статус

<Черновик | Актуален>

## Последнее обновление

YYYY-MM-DD
```

## Статус

Актуален
