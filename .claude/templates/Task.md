# Шаблон: Task

## Назначение

Шаблон задачи для системы `tasks/`. Файл задачи проходит все стадии жизненного цикла ([docs/architecture/workflow.md](../../docs/architecture/workflow.md)) и накапливает историю: замечания ревью и отчёт исполнителя добавляются в этот же файл.

## Содержание

```markdown
# TASK-NNN: <Название задачи>

## Тип

<feature | bugfix | docs | refactor | test | chore>

## Эпик

<EPIC-NNN или «вне эпика»>

## Цель

<Какой результат нужен и зачем.>

## Контекст

<Ссылки на документы, ADR, связанные задачи. Ограничения.>

## Scope

### Входит

- <что входит>

### Не входит

- <что явно не входит>

## Критерии приёмки

- [ ] <проверяемый критерий>
- [ ] <проверяемый критерий>

## Затрагиваемые модули и документы

- <пути>

## Definition of Ready

- [ ] Цель и результат сформулированы
- [ ] Критерии приёмки определены
- [ ] Затрагиваемые модули/документы указаны
- [ ] Ограничения и зависимости указаны

## План реализации

<Заполняется исполнителем ДО написания кода; реализация начинается только
после утверждения плана человеком (запись об утверждении — в Истории).
Шаги, затрагиваемые файлы, порядок проверок.>

## История

<Записи о переходах между стадиями, замечания ревью, причины блокировок.
Формат записи: YYYY-MM-DD — <кто> — <что произошло>.>

## Отчёт о выполнении

<Заполняется исполнителем по структуре из CLAUDE.md:
задача, что сделано, изменённые файлы, как проверялось,
обновлённая документация, Open Questions, рекомендации.>
```
