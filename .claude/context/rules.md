# Контекст: Операционные правила для агентов

## Назначение

Сжатый свод обязательных правил для AI-агента на каждый запуск. Источники истины — [CONSTITUTION.md](../../CONSTITUTION.md) и [CLAUDE.md](../../CLAUDE.md); при расхождении приоритет у них.

## Содержание

### Всегда

1. Прочитать Конституцию, контекст (`.claude/context/`) и файл задачи до начала работы.
2. Работать строго в scope задачи из `tasks/ready/`, соответствующей DoR.
3. **Сначала план, потом код:** записать план реализации в файл задачи и дождаться утверждения человеком; без утверждённого плана реализация не начинается.
4. Вести задачу по стадиям: ready → in-progress → review (или blocked) — перемещением файла с записью в истории задачи.
5. Обновлять документацию в том же PR, что и изменения; у каждого кодового модуля — актуальный README.
6. Прогонять `make verify` перед сдачей задачи; не сдавать задачу с падающими проверками.
7. Оформлять изменения по git-процессу: ветка, Conventional Commits, PR по шаблону, чек-лист PR.
8. Формировать отчёт о выполнении в файле задачи; исправлять замечания ревью до merge.
9. Фиксировать любую неопределённость в Open Questions.

### Никогда

1. Не принимать архитектурные решения; не создавать ADR без явного поручения.
2. Не добавлять технологии вне [stack.md](stack.md).
3. Не изменять структуру репозитория и Конституцию.
4. Не расширять scope («заодно исправил»).
5. Не коммитить в `main` напрямую; не сливать свой PR без одобрения ревью.
6. Не придумывать требования — при их отсутствии остановиться и зафиксировать вопрос.
7. Не удалять чужие ADR, отчёты и записи истории задач.

### При блокировке

Переместить задачу в `tasks/blocked/`, записать причину и требуемое решение, остановиться.
