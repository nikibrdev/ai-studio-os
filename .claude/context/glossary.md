# Контекст: Глоссарий

## Назначение

Единая терминология проекта. Все документы и код используют термины в значениях, определённых здесь. Полные описания сущностей — [docs/architecture/domain-model.md](../../docs/architecture/domain-model.md).

## Содержание

### Термины процесса

| Термин | Значение |
| --- | --- |
| **Роль** | Набор обязанностей в процессе разработки (PM, Developer, QA, Reviewer, Architect), не привязанный к исполнителю |
| **Исполнитель** | Человек или AI-агент, выполняющий роль |
| **Задача (Task)** | Единица работы с формализованным жизненным циклом; markdown-файл по шаблону Task |
| **Эпик (Epic)** | Крупная цель, декомпозируемая на задачи |
| **Workflow** | Определение процесса прохождения задачи: шаги и правила переходов |
| **Шаг workflow (Workflow Step)** | Отдельный шаг процесса с ответственной ролью и условиями входа/выхода |
| **State machine задачи** | Канонический набор состояний и переходов задачи — [docs/architecture/state-machine.md](../../docs/architecture/state-machine.md) |
| **DoR** | Definition of Ready — критерии готовности задачи к работе |
| **DoD** | Definition of Done — критерии завершённости задачи |
| **Open Questions** | Зафиксированные вопросы, требующие решения человека; способ работы с неопределённостью вместо самостоятельных решений агента |
| **Спецификация (Specification)** | Полное ТЗ модуля в `docs/specifications/<слой>/`; вход для задач реализации. README модуля — краткая версия для разработчика |
| **Decision Required** | Статус заготовки ADR: варианты описаны, решение архитектора не принято |

### Термины исполнения

| Термин | Значение |
| --- | --- |
| **Агент** | AI-система, исполняющая роль через адаптер платформы |
| **Адаптер агента** | Компонент в `agents/`, подключающий конкретного AI-провайдера к контракту Agent |
| **Инструмент (Tool)** | Действие во внешней среде, доступное агенту через Tool Layer (`tools/`) |
| **Исполнение (Execution)** | Один запуск исполнителя для выполнения задачи или шага workflow |
| **Результат (Result)** | Неизменяемый итог завершённого исполнения: статус, отчёт, артефакты |
| **Артефакт (Artifact)** | Неизменяемый объект, произведённый в ходе работы: отчёт, диф, лог |
| **Память (Memory)** | Накапливаемые знания проекта (`memory/`), с v0.7 — с векторным поиском |

### Термины системы

| Термин | Значение |
| --- | --- |
| **Событие (Event)** | Неизменяемый зафиксированный факт; единственный способ междоменного взаимодействия |
| **Шина событий (Event Bus)** | Контракт доставки событий подписчикам и ведения журнала |
| **Ядро (Core)** | Доменная логика платформы (`internal/`), независимая от инфраструктуры и AI-провайдеров |
| **Платформенный слой (platform)** | Контракты инфраструктурных абстракций (`internal/platform`): EventBus, Agent, Tool, MemoryProvider, RepositoryProvider; не язык домена (ADR-015) |
| **Язык домена (domain/shared)** | Словарные типы предметной области (`internal/domain/shared`): Role, TaskState и др. |
| **Модуль** | Единица ядра с единственной ответственностью и владением своими сущностями |
| **Порт** | Интерфейс, который модуль ядра требует от инфраструктуры |
| **Проекция** | Производное представление данных для чтения, перестраиваемое из событий; не источник истины |
| **Управляемый проект** | Проект, разработкой которого управляет платформа (`projects/`) |
| **Сессия (Session)** | Аутентифицированный сеанс работы пользователя с платформой |
| **ADR** | Architecture Decision Record — зафиксированное архитектурное решение (`docs/adr/`) |
| **Конституция** | [CONSTITUTION.md](../../CONSTITUTION.md) — главный свод правил проекта, изменяемый только через ADR |
