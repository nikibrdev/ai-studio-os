# Контекст: Видение

## Назначение

Краткое видение продукта для быстрой загрузки контекста AI-агентом перед работой. Источник истины — [VISION.md](../../VISION.md); этот файл — сжатая выжимка, не дублирует его содержание.

## Содержание

### Что мы строим

**AI Studio OS** — операционная система для инженерной команды, а не «система агентов»: она умеет запускать любые типы Исполнителей (Executor) — Claude Code, Codex, OpenHands, Aider, Cline, человека, в будущем — собственные модели. Ядро не знает, кто именно выполняет работу; оно знает только, что Executor принимает Context и возвращает Result.

### Ключевые тезисы

1. Роль отделена от исполнителя: процесс одинаков для человека и любого Executor'а.
2. Ядро не приобретает зависимостей от конкретного AI-провайдера — агент подключается через адаптер, не встраивается в ядро.
3. Процесс прозрачен: задачи, события, отчёты и решения фиксируются и наблюдаемы.
4. Качество выше скорости; документация первична (Architecture → Specification → Implementation).
5. Архитектура обслуживает видение, а не наоборот — каждый новый модуль оценивается на приближение к эталонному сценарию ([golden-path.md](../../docs/architecture/golden-path.md)).

### Что продуктом НЕ является

- Не «автопилот», бесконтрольно пишущий код: человек всегда видит и контролирует процесс.
- Не обёртка над одной AI-моделью.
- Не замена git/GitHub — платформа встраивается в стандартный процесс разработки.
