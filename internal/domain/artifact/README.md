# Модуль: internal/domain/artifact

## Назначение

Сущность Artifact — самостоятельный Aggregate Root, долговременный инженерный результат, который платформа способна создавать, сохранять и переиспользовать независимо от процесса, в рамках которого он появился ([полная спецификация](../../../docs/specifications/domain/artifact.md), статус Reference; [ADR-016](../../../docs/adr/ADR-016-artifact-aggregate-root.md)).

## Содержание

### Ответственность

- Тип `Artifact` — сущность с Metadata (Identifier, ProjectID, Type, Origin, Author, CreatedAt, опционально ProducedBy) и Payload; инварианты применяются в коде, не только описаны.
- Lifecycle: `Draft → Published → Archived`, включая прямой `Draft → Archived`; переходы необратимы.
- Команды как методы: `New` (Create), `UpdateDraft`, `Publish`, `Archive` — каждая возвращает соответствующее доменное событие (`Created`/`Published`/`Archived`) как значение, без публикации через шину (см. «Не входит» ниже).

### Отличие от других доменных модулей

В отличие от `internal/domain/{task,project,workflow}`, этот пакет реализует настоящую доменную логику (инварианты, проверяемые в коде), а не только контракты-интерфейсы. Он не выставляет Commands/Queries-интерфейсы в стиле `task`/`project`: это описывало бы путь записи для ещё не выбранного механизма персистентности, для которого пока нет конкретного потребителя (Infrastructure/Application для Artifact не начаты).

### Зависимости

- Разрешено: только стандартная библиотека.
- Запрещено: `internal/platform`, другие доменные модули, application, infrastructure ([ADR-015](../../../docs/adr/ADR-015-internal-layering.md)).

### События

Domain Events возвращаются командами как значения (`Created`, `Published`, `Archived`) — публикация через Event Bus (`internal/platform`) не входит в этот пакет.

### Ограничения

- Полный перечень значений `Origin` — открытый вопрос спецификации; тип оставлен открытой строкой, а не закрытым перечислением.
- Кардинальность связи с породившим Execution, состав Metadata сверх обязательного минимума — см. Open Questions спецификации.
