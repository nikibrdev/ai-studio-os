# Модуль: internal/domain/project

## Назначение

Реестр проектов, которыми управляет платформа: создание, подключение репозиториев, архивирование ([domain-model.md](../../../docs/architecture/domain-model.md), «Project»).

## Содержание

### Ответственность

- `Registry` — жизненный цикл проекта: Created → Active → Archived (архив неизменяем).
- Владение данными: Project, назначения исполнителей ролей в проекте (контракт назначений — Domain Layer).

### Зависимости

- Разрешено: stdlib.
- Запрещено: другие доменные модули, application, infrastructure, драйверы.

### События

События проекта (создание, подключение репозитория, архивирование) будут определены при реализации модуля (Domain Layer); каталог — [events.md](../../../docs/architecture/events.md).

### Ограничения

Формат подключения репозиториев и содержимое `projects/` — Decision Required ([ADR-013](../../../docs/adr/ADR-013-managed-projects.md)).
