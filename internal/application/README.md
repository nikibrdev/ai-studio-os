# Слой: internal/application

## Назначение

Application Layer — сценарии использования (use cases), координирующие доменные модули: приём команд, оркестрация операций, построение проекций для чтения. Заполняется эпиком Application Layer (после Domain Layer).

## Содержание

### Ответственность (целевая)

- Сценарии команд: «создать задачу», «перевести задачу», «назначить исполнителя».
- Проекции для чтения (Dashboard/API) из событий ([ADR-014](../../docs/adr/ADR-014-module-interaction.md)).
- Транзакционные границы вокруг доменных операций.

### Зависимости

- Разрешено: `internal/domain/*` (публичные контракты), `internal/platform` (контракты абстракций), `pkg/`.
- Запрещено: `internal/infrastructure` (только через контракты platform), `apps/`, доменные правила (они в domain).

### События

Подписывается на события доменных модулей для построения проекций; собственных событий не публикует.

### Текущее состояние

Пусто (только этот README) — код появится в эпике Application Layer.
