# Domain — предметная область

## Назначение

Единый язык предметной области (Ubiquitous Language, DDD) и границы контекстов (Bounded Contexts) AI Studio OS — до того, как понятия становятся Go-структурами. Введено архитектором проекта 2026-07-20, перед началом Domain Layer (v0.3, EPIC-003): сначала понятия бизнеса, затем модели.

### Отличие от других источников терминов

- **Здесь** — авторитетное определение понятия как бизнес-концепции: зачем оно существует, кто им пользуется, как соотносится с другими понятиями.
- **[.claude/context/glossary.md](../../.claude/context/glossary.md)** — краткий быстрый справочник терминов для AI-агента при чтении кода; ссылается сюда, не дублирует.
- **[docs/specifications/domain/](../specifications/domain/)** — техническая спецификация конкретного Go-пакета (требования, инварианты, события, Acceptance Criteria); реализует понятие отсюда, но не определяет его заново.
- **[docs/architecture/domain-model.md](../architecture/domain-model.md)** — как понятия отражены в текущей архитектуре (владение данными, состояния, связи между модулями); технический слой поверх языка, определённого здесь.

Порядок работы: язык (здесь) → границы контекста (здесь) → спецификация модуля (`docs/specifications/domain/`) → реализация.
