# Правила документации

## Назначение

Определяет, как в AI Studio OS создаётся и сопровождается документация. Documentation First — один из базовых принципов проекта: документ создаётся раньше кода.

## Содержание

### Единая структура документа

Каждый markdown-документ проекта имеет структуру:

```markdown
# <Заголовок>

## Назначение
<Зачем существует документ и для кого он>

## Содержание
<Основное содержимое; допускаются подразделы>

## Статус
<Актуален | Черновик | Заглушка | Superseded>

## Последнее обновление
<YYYY-MM-DD>
```

### Где какая документация живёт

| Место | Содержание |
|---|---|
| `docs/architecture/` | Архитектура системы |
| `docs/adr/` | Архитектурные решения (ADR) |
| `docs/development/` | Процессы разработки |
| `docs/api/` | Документация API (с v0.2+) |
| `docs/operations/` | Эксплуатация (позже) |
| `docs/roadmap/` | Детализация roadmap |
| `.claude/` | Документы, управляющие поведением AI-агентов |
| Корень | README, CONSTITUTION, CLAUDE, ROADMAP, CHANGELOG |

Правило разграничения: `docs/` описывает систему; `.claude/` управляет агентами.

### README каждого модуля (обязательно)

Каждый кодовый модуль (пакет в `internal/`, слой, приложение в `apps/`, адаптер) обязан иметь `README.md` с разделами: **назначение**, **зависимости** (что разрешено/запрещено), **события** (что публикует/потребляет), **ответственность** (владение данными). README обновляется в том же PR, что и изменение модуля. Пример: [internal/domain/task/README.md](../../internal/domain/task/README.md).

### Правила ведения

1. **Один факт — один источник истины.** Дублирование запрещено; остальные документы ссылаются на источник.
2. **Документация меняется вместе с кодом** — в том же PR.
3. **Каждое изменение документа** обновляет поле «Последнее обновление».
4. **Ссылки** — относительные, на файлы внутри репозитория.
5. **Язык** — русский; код и идентификаторы — английский.
6. **Стиль** — профессиональный, без воды; короткие абзацы, списки, таблицы.
7. **Устаревшие документы** не удаляются молча: помечаются статусом Superseded со ссылкой на замену.

### ADR

Архитектурные решения фиксируются в `docs/adr/` по шаблону [ADR-000-template.md](../adr/ADR-000-template.md). Нумерация — сквозная (`ADR-001`, `ADR-002`, ...). Когда создавать ADR — см. [CONSTITUTION.md](../../CONSTITUTION.md) и [CLAUDE.md](../../CLAUDE.md).

### Decision Required
