# Стандарт кодирования

## Назначение

Единые правила написания кода в AI Studio OS для людей и AI-агентов. Обязателен для всего кода начиная с v0.2; конкретные конфигурации линтеров будут добавлены при появлении кода.

## Содержание

### Общие правила

1. Код пишется для читателя: ясные имена, короткие функции, очевидный поток управления.
2. KISS и DRY: без преждевременных абстракций и без дублирования логики.
3. Interface First: публичное поведение модуля описывается интерфейсом до реализации.
4. Минимум магии: без скрытых side effects, неявной инициализации и рефлексии без необходимости.
5. Комментарий объясняет «почему», а не «что»; устаревшие комментарии удаляются вместе с кодом.
6. Язык идентификаторов и комментариев в коде — английский; документация — на русском.

### Go (backend)

1. Форматирование — **gofumpt** (обязательно; строже gofmt), импорты — `goimports` ([ADR-009](../adr/ADR-009-toolchain.md)).
2. Именование и стиль — по [Effective Go](https://go.dev/doc/effective_go) и стандартным соглашениям сообщества.
3. Ошибки: возвращаются явно, оборачиваются с контекстом (`fmt.Errorf("...: %w", err)`); паника — только при невозможности продолжать работу процесса.
4. Пакеты: маленькие, с единственной ответственностью; имя пакета — короткое существительное.
5. Доменная логика — в `internal/`; зависимость от инфраструктуры — только через интерфейсы.
6. Контекст (`context.Context`) — первым аргументом во всех операциях с вводом-выводом.
7. Конкурентность — только при доказанной необходимости; каналы и горутины не используются «на всякий случай».
8. Статический анализ — `go vet` и **golangci-lint** (конфигурация — `.golangci.yml` в корне; [ADR-009](../adr/ADR-009-toolchain.md)).

### TypeScript / Next.js (frontend)

1. Строгий TypeScript (`strict: true`); тип `any` запрещён без обоснования в коде.
2. Компоненты — маленькие, с единственной ответственностью; логика выносится из компонентов представления.
3. Взаимодействие с backend — только через API-клиент; прямых обращений к инфраструктуре нет.
4. Менеджер пакетов — **pnpm**, фреймворк — **Next.js 15** ([ADR-009](../adr/ADR-009-toolchain.md)); конфигурации Prettier/ESLint фиксируются при создании приложения dashboard.

### Правила для AI-агентов

1. Агент следует существующему стилю файла, который редактирует.
2. Агент не переформатирует и не рефакторит код вне scope задачи.
3. Сгенерированный код проходит те же проверки и ревью, что и написанный человеком.
