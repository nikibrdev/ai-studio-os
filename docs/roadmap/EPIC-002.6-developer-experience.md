# EPIC-002.6: Developer Experience (DX)

## Цель

Подготовить среду разработки так, чтобы новый участник — человек или AI-агент — получал рабочее окружение по принципу «клонируй → одна команда → проект готов», а качество контролировалось ещё до коммита. Закрепляет фундамент для v0.3 (Domain Layer): после этого эпика стоимость любого следующего изменения ниже.

## Контекст

Предложен архитектором проекта 2026-07-19, полный scope и порядок утверждены им же 2026-07-20 после приёмки EPIC-002.5. Опирается на готовые `make verify`, `.golangci.yml`, `scripts/verify-docs.sh`, защищённый `main` (EPIC-002.5). До EPIC-003 (Domain Layer) не переходим.

## Scope

### Входит

1. Pre-commit / pre-push хуки по уже принятому уровню проверок ([решение](../../engineering/decisions/2026-07-19-quality-gates.md)).
2. Минимальный Dev Container.
3. `.editorconfig`.
4. Рекомендуемые настройки VS Code: `.vscode/extensions.json` и `.vscode/settings.json`.
5. `CONTRIBUTING.md` — пошаговая инструкция для нового разработчика.
6. Проверка: новый разработчик клонирует проект и запускает проверки без дополнительных ручных действий.

### Не входит

- Domain Layer и прикладной код; CI (это EPIC-002.5, уже сделано); деплой.

## Решения по вопросам (приняты архитектором)

1. Devcontainer — делаем сразу, минимальный (состав в TASK-024); отвечает только за среду разработки (2026-07-19).
2. Pre-commit — быстрый набор (fmt + lint + vet); полный `make verify` — на pre-push и в CI, без исключений (2026-07-19).
3. Инструкция для новых участников — `CONTRIBUTING.md` в корне (стандартное соглашение open-source), а не `docs/development/onboarding.md` (2026-07-20).
