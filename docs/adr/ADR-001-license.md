# ADR-001: Лицензия проекта

## Статус

**Принято** (решение владельца проекта, 2026-07-19)

## Дата

2026-07-19

## Контекст

Проект публикуется как open-source (public-репозиторий, [решение](../../engineering/decisions/2026-07-19-github-repository.md)); placeholder «All rights reserved» юридически делал его закрытым и блокировал внешние контрибуции. Лицензию требовалось выбрать до первого push.

## Рассмотренные варианты

1. **MIT** — максимально простая, но без явной патентной защиты.
2. **Apache License 2.0** — явный патентный грант, стандарт инфраструктурных проектов.
3. **AGPL-3.0** — защита от закрытых SaaS-форков ценой отпугивания корпоративных пользователей.

## Решение

Принята **Apache License 2.0**.

Обоснование:

- явный патентный грант (Grant of Patent License, §3);
- лучше подходит для инфраструктурных и инженерных проектов;
- стандарт для платформ, SDK и инструментов;
- если AI Studio OS станет основой коммерческих решений — более чёткие юридические гарантии, чем MIT.

## Последствия

### Положительные

- Патентная защита контрибьюторов и пользователей; предсказуемые условия для корпоративного использования.
- Совместимость с большинством open-source и коммерческих сценариев.

### Отрицательные

- Формальнее MIT: изменённые файлы должны нести пометки об изменениях (§4b); при появлении attribution-требований — файл NOTICE.

### Влияние на существующие документы и код

[LICENSE](../../LICENSE) — полный текст Apache-2.0; [README.md](../../README.md) — раздел «Лицензия»; будущий CONTRIBUTING — условия приёма контрибуций (§5: контрибуции принимаются под той же лицензией).
