# ADR-007: Исполнители ролей Project Manager и QA Engineer в MVP

## Статус

**Decision Required** — определяет объём реализации v0.2 (Project Management) и v0.4 (QA Engine): что строится для человека (UI/процессы), а что для агента (промпты, адаптеры). Требуется решение владельца продукта: это вопрос целевого пользовательского опыта MVP.

## Дата

2026-07-19 (заготовка)

## Проблема

MVP включает три роли; Developer по умолчанию — Claude Code. Кто по умолчанию исполняет роли Project Manager и QA Engineer в MVP — человек, агент или гибрид — не определено ([agents.md](../architecture/agents.md)).

## Контекст

- Роль отделена от исполнителя; процесс одинаков для обоих ([CONSTITUTION.md](../../CONSTITUTION.md)).
- До v0.6 нет Dashboard — человеку-PM работать неудобно (файлы и git напрямую).
- Качество автономного PM/QA-агента без обкатанных промптов и памяти (v0.7) не гарантировано.
- Definition of Done требует контроля качества — чьего именно, влияет на доверие к результату.

## Возможные варианты

### Вариант 1: Обе роли — человек (агенты только Developer)

- **Плюсы:** контроль качества и постановки задач в руках человека; минимум реализации в v0.2/v0.4; безопасный старт.
- **Минусы:** платформа менее «мультиагентна» на практике; узкое место — доступность человека.

### Вариант 2: Обе роли — агенты с подтверждением человеком ключевых решений

- **Плюсы:** демонстрирует ценность платформы; человек — только на контрольных точках (утверждение DoR, приём Done).
- **Минусы:** нужны промпты/адаптеры для двух дополнительных ролей уже в MVP; риск качества без памяти проекта.

### Вариант 3: Гибрид — PM человек, QA агент (или наоборот)

- **Плюсы:** постановка задач остаётся за человеком (наибольший риск ошибок), проверка автоматизируется (наиболее формализуемая часть).
- **Минусы:** асимметрия процессов; всё равно нужен агентский контур для одной из ролей.

## Влияние на систему

Scope v0.2 и v0.4 ([ROADMAP.md](../../ROADMAP.md)); приоритет Dashboard (v0.6) для человеко-ролей; описания ролей `.claude/agents/`; требования к контрольным точкам процесса ([workflow.md](../architecture/workflow.md)).
