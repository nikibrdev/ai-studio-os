# ADR-009: Toolchain и компоновка кода

## Статус

**Принято** (решение архитектора проекта, 2026-07-19)

## Дата

2026-07-19

## Контекст

Стек был утверждён на уровне технологий ([stack.md](../../.claude/context/stack.md)); версии, инструменты качества и компоновка кода требовали фиксации до первой строки кода.

## Рассмотренные варианты

Рассматривались: единый `go.mod` против Go workspace; npm/yarn/pnpm; составы линтеров (см. историю вопроса в описании Decision Required этапа Architecture v0.2).

## Решение

Зафиксировано:

| Позиция | Решение |
|---|---|
| Backend | **Go 1.24** |
| Frontend | **Next.js 15** |
| Package Manager (frontend) | **pnpm** |
| Linter (Go) | **golangci-lint** |
| Formatter (Go) | **gofumpt** |

Дополнительно зафиксировано при оформлении (простейший вариант, необходимый для старта EPIC-002; пересмотр — новым ADR):

- **Компоновка Go: единый `go.mod` в корне репозитория**; границы модулей обеспечиваются каталогом `internal/`, правилами [module-boundaries.md](../architecture/module-boundaries.md) и линтером.
- **Имя Go-модуля: `ai-studio-os`** (локальное; при публикации на GitHub заменяется на полный путь — механическая замена, оформляется задачей).

Не зафиксировано (не блокирует текущие этапы, решается при старте соответствующей работы): точная версия Node.js и конфигурация ESLint/Prettier (при создании `apps/dashboard`), unit-фреймворк frontend (v0.4), физическое размещение адаптеров инфраструктуры (этап Infrastructure).

## Последствия

### Положительные

- Единые версии и инструменты для людей и агентов; воспроизводимые проверки.
- gofumpt — строже gofmt: единообразный стиль без обсуждений.
- Единый `go.mod` — минимальная сложность сборки монорепозитория.

### Отрицательные

- Границы модулей при едином `go.mod` не принудительны на уровне компилятора — контроль на ревью и линтером.
- Обновление версий Go/Next.js — осознанным решением (chore-задачей), не автоматически.

### Влияние на существующие документы и код

`go.mod` (EPIC-002, TASK-001); [coding-standard.md](../development/coding-standard.md) и [stack.md](../../.claude/context/stack.md) — обновлены; Makefile — цели fmt/lint/build наполняются по мере появления кода.
