# ADR-010: Язык документации

## Статус

**Принято** (решение владельца проекта, 2026-07-23) — **Вариант 1**.

## Дата

2026-07-19 (заготовка); принято — 2026-07-23, при открытии v1.0 First Public MVP

## Проблема

Вся документация ведётся на русском ([CONSTITUTION.md](../../CONSTITUTION.md)). Для публикации как международный open-source проект (v1.0) это сужает аудиторию пользователей и контрибьюторов. Нужна ли английская версия и в каком объёме — не решено.

## Контекст

- Документация объёмная и меняется часто (Documentation First) — перевод «вручную вслед» будет отставать.
- AI-агенты работают с любым языком; вопрос касается людей.
- Синхронизация двух языковых версий — постоянные издержки и риск расхождений (нарушение «один факт — один источник»).

## Возможные варианты

### Вариант 1: Только русский до v1.0, решение о переводе — перед публичным релизом

- **Плюсы:** нет издержек сейчас; фокус на содержании.
- **Минусы:** перевод всего корпуса перед релизом — большой разовый объём; ранние внешние контрибьюторы отсечены.

### Вариант 2: Двуязычная документация (русский — основной, английский — перевод)

- **Плюсы:** открытость с самого начала.
- **Минусы:** двойное сопровождение каждого документа; расхождения версий почти неизбежны.

### Вариант 3: Английский — основной для публичных документов (README, CONSTITUTION), русский — рабочий

- **Плюсы:** витрина проекта международная при умеренных издержках.
- **Минусы:** смешение языков в одном репозитории; критерий «публичности» документа размыт.

## Решение

Принят **Вариант 1**: весь корпус документации (240+ файлов на момент решения) остаётся на русском для v1.0. Полный перевод — отдельное, самостоятельное по объёму предприятие (двойное сопровождение каждого документа, риск расхождений — тот же класс проблемы, что и Вариант 2 явно называет своим минусом), не вмещается в объём эпика v1.0 без отдельного решения о его старте. Решение о переводе (частичном или полном) — по факту появления реального англоязычного контрибьютора или пользователя, тот же принцип «не строить сверх доказанной потребности», что и во всех предыдущих решениях этой сессии (ADR-018 — эмбеддинги, ADR-012 — аутентификация).

## Влияние на систему

Правила [documentation.md](../development/documentation.md) — не меняются. Процесс релиза (чек-лист Release) — v1.0 публикуется на русском без блокировки. Структура `docs/` (языковые каталоги) — не вводится.
