# Модель агентов

## Назначение

Определяет концепцию AI-агентов в AI Studio OS: как роли отделены от исполнителей, как достигается независимость от AI-провайдеров и по каким правилам агенты работают в системе. Реализация — начиная с v0.3 (Developer Engine) и v0.9 (Multi Agent).

## Содержание

### AI Studio OS — операционная система, агенты — плагины

Видение проекта ([VISION.md](../../VISION.md)) переформулирует эту модель жёстче, чем просто «роль ≠ исполнитель»: **AI Studio OS — операционная система, умеющая запускать любые типы Исполнителей (Executor)** — Claude Code, Codex, OpenHands, Aider, Cline, человека, в будущем — собственные модели. Ядро не знает, кто именно выполняет работу; оно знает только, что существует Executor, принимающий Context и возвращающий Result ([ubiquitous-language.md](../domain/ubiquitous-language.md)). Агент — лишь один из видов Executor'а: AI-плагин, а не центральное понятие платформы.

Эта абстракция закреплена в коде: контракт `platform.Executor` (`Accept`/`Artifacts`/`Status`/`Finish`, [ADR-005](../adr/ADR-005-executor-contract.md), принято) — не как временная заглушка, а как соответствие видению: ядро не должно приобретать зависимостей от конкретного провайдера или формата обмена раньше, чем это станет необходимо; форма `ExecutorTask`/`Artifact`/`ExecutionStatus` остаётся абстрактной до Domain Layer.

### Ключевая идея: роль ≠ исполнитель

Платформа оперирует **ролями** (Project Manager, Developer, QA Engineer). Исполнителем роли может быть:

- человек;
- AI-агент (Claude Code, OpenAI Codex, OpenHands, Aider, Cline и др.).

Процесс, права и обязанности определяются ролью и не зависят от того, кто её исполняет. Обязанности ролей описаны в [.claude/agents/](../../.claude/agents/).

### Независимость от провайдера

Ядро системы не знает о конкретных AI-моделях. Подключение агента выполняется через **адаптер** в каталоге `agents/`, реализующий единый контракт (Interface First):

- принять назначенную задачу и контекст;
- выполнять работу в рамках роли и правил проекта;
- сообщать о ходе работы событиями;
- вернуть результат и отчёт.

Добавление нового провайдера — это добавление адаптера, без изменения ядра. Claude Code — реализация роли Developer по умолчанию; его инструкция — [CLAUDE.md](../../CLAUDE.md).

### Правила работы агентов

1. Агент действует строго в рамках роли и scope задачи.
2. Агент не принимает архитектурных решений; неопределённость фиксируется в Open Questions.
3. Результат работы агента проходит стандартный процесс ревью.
4. Действия агента наблюдаемы: события, отчёты, изменения — всё фиксируется.
5. Возможности агента ограничиваются слоем инструментов (см. [tools.md](tools.md)).

### MVP-роли

| Роль | Обязанности (кратко) | Исполнитель по умолчанию |
| --- | --- | --- |
| Project Manager | Подготовка и приоритизация задач, управление процессом | не определён |
| Developer | Реализация задач, оформление изменений | Claude Code |
| QA Engineer | Контроль качества, тестирование | не определён |
