# Событийная модель

## Назначение

Определяет принципы событийного взаимодействия модулей AI Studio OS. Конкретные события описаны в каталоге [events.md](events.md); механизм доставки и формат схем — Decision Required ([ADR-002](../adr/ADR-002-event-delivery.md)).

## Содержание

### Принципы

1. **События — единственный способ междоменного взаимодействия.** Модуль не вызывает логику другого модуля напрямую; он публикует событие о факте, который произошёл. (Допустимость дополняющих read-контрактов — [ADR-014](../adr/ADR-014-module-interaction.md).)
2. **Событие описывает факт, а не команду.** Именование — в прошедшем времени: TaskCreated, ReviewCompleted. Команды (запросы на действие) отделяются от событий и адресуются модулю-владельцу.
3. **События неизменяемы** (Immutable Events, [engineering-principles.md](engineering-principles.md)): после публикации событие не редактируется и не удаляется; изменение структуры — новая версия схемы.
4. **События — часть публичного контракта.** Схема события принадлежит модулю-источнику и документируется в момент создания; изменение схемы — через ADR.
5. **Подписчики независимы.** Публикующий модуль не знает своих подписчиков; добавление подписчика не требует изменения публикующей стороны (расширяемость без изменения ядра).
6. **Журнал обязателен.** Все события фиксируются в журнале: аудит, история, перестроение проекций.
7. **Минимум магии.** Маршрутизация событий явная и читаемая; никакой автоматической регистрации по соглашениям.

### Каталог событий

Полный перечень событий с источниками, получателями, данными и последствиями — [events.md](events.md). Соответствие событий переходам жизненного цикла задачи — [state-machine.md](state-machine.md).

### Принятые решения

- [ADR-002](../adr/ADR-002-event-delivery.md) — **In-Memory Event Bus**: внутрипроцессная доставка; интерфейс шины неизменен при будущей замене реализации (Redis Streams / NATS); журнал — подписчик, сохраняющий события в PostgreSQL.
- [ADR-014](../adr/ADR-014-module-interaction.md) — проекции строятся только из событий; синхронные read-контракты между модулями не вводятся.
