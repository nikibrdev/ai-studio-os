# Слой инструментов

## Назначение

Определяет концепцию Tool Layer — унифицированного слоя инструментов, через который агенты взаимодействуют с внешним миром. Реализация запланирована на v0.8 Tool Layer.

## Содержание

### Зачем нужен слой инструментов

Агенты не должны обращаться к инфраструктуре напрямую и произвольно. Слой инструментов решает три задачи:

1. **Унификация** — один и тот же инструмент (например, «создать PR») одинаково доступен любому агенту независимо от провайдера.
2. **Контроль** — возможности агента ограничены явным набором инструментов; всё, что не разрешено инструментом, агенту недоступно.
3. **Расширяемость** — новый инструмент добавляется в `tools/` декларативно, без изменения ядра и без изменения агентов.

### Принципы

1. Инструмент — это контракт (Interface First): имя, назначение, входные и выходные данные, ограничения. Контракт документируется в момент создания.
2. Инструмент не содержит доменной логики платформы — только действие во внешней среде (git, файлы, запуск тестов и т.п.).
3. Набор инструментов, доступных агенту, определяется его ролью и задачей, а не самим агентом.
4. Вызовы инструментов наблюдаемы: фиксируются как события с параметрами и результатом.
5. Минимум магии: никакого автоматического обнаружения инструментов по соглашениям — только явная регистрация.

### Ожидаемые группы инструментов (ориентир, не контракт)

- Работа с git и GitHub (ветки, коммиты, PR).
- Работа с файлами управляемого проекта.
- Запуск проверок (сборка, линтеры, тесты, Playwright).
- Работа с системой задач и памятью.

Точный перечень будет определён при проектировании v0.8.

### Открытые вопросы

Концептуальный контракт Tool описан в [interfaces.md](interfaces.md). Не определены (проектирование v0.8):

- Формат описания инструмента (декларация, схема параметров) не определён.
- Механизм регистрации инструментов и их сопоставления ролям не спроектирован.
- Модель безопасности вызова инструментов (права, песочница, подтверждение опасных операций) не определена.
- Соотношение Tool Layer и собственных инструментов конкретных агентов (например, встроенных инструментов Claude Code) требует проработки в v0.8.
