# Roadmap AI Studio OS

## Назначение

Определяет этапы развития проекта от фундамента до стабильного релиза. Содержание каждой версии уточняется отдельными эпиками и задачами; настоящий документ фиксирует цели и границы этапов.

## Содержание

### v0.1 Foundation — фундамент

- Структура репозитория.
- Базовая документация: Конституция, README, Roadmap, Changelog.
- Архитектурная документация и документация процессов разработки.
- Система задач (`tasks/`) и шаблоны (`.claude/`).
- Роли агентов описаны на уровне обязанностей (без реализации).

**Результат:** репозиторий готов к началу разработки; правила и процессы зафиксированы.

### v0.2 Project Management — управление проектами

- Модель проекта и задачи.
- Жизненный цикл задачи в системе.
- Роль Project Manager (создание, декомпозиция, приоритизация задач).

**Результат:** задачи создаются и управляются внутри платформы.

### v0.3 Developer Engine — движок разработчика

- Роль Developer на базе Claude Code.
- Выполнение задач агентом по процессу из [CLAUDE.md](CLAUDE.md).
- Оформление изменений (ветки, PR) через GitHub.

**Результат:** агент-разработчик выполняет подготовленные задачи.

### v0.4 QA Engine — движок контроля качества

- Роль QA Engineer.
- Проверка результатов работы Developer.
- Тестирование, включая end-to-end сценарии (Playwright).

**Результат:** каждая задача проходит контроль качества.

### v0.5 Workflow — рабочий процесс

- Прохождение задачи по цепочке ролей: PM → Developer → QA.
- Событийная модель переходов между стадиями.
- Обработка блокировок и возвратов на доработку.

**Результат:** полный цикл задачи выполняется по формализованному процессу.

### v0.6 Dashboard — веб-интерфейс

- Интерфейс на Next.js: проекты, задачи, статусы, отчёты агентов.
- Наблюдаемость процесса для человека-руководителя.

**Результат:** состояние платформы видно и управляемо через веб-интерфейс.

### v0.7 Memory — память

- Память агентов и знания проекта (`memory/`).
- Векторный поиск на Qdrant.

**Результат:** агенты используют накопленный контекст проекта.

### v0.8 Tool Layer — слой инструментов

- Унифицированный слой инструментов (`tools/`), доступных агентам.
- Подключение инструментов без изменения ядра.

**Результат:** возможности агентов расширяются инструментами декларативно.

### v0.9 Multi Agent — мультиагентность

- Подключение альтернативных агентов (OpenAI Codex, OpenHands, Gemini и др.) через адаптеры.
- Выбор исполнителя роли: человек или конкретный агент.

**Результат:** платформа независима от конкретной AI-модели на практике, а не только в архитектуре.

### v1.0 Stable — стабильный релиз

- Стабилизация API и контрактов.
- Полная документация для пользователей и контрибьюторов.
- Публичный релиз.

**Результат:** платформа готова к использованию внешними командами.
